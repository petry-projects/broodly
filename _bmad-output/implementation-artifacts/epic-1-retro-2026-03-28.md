## Challenges & Lessons Learned

### 1. Expo SDK Canary Versions
**Issue:** `create-expo-app@3.5.3` installed canary versions of Expo SDK 55 packages (`55.0.10-canary-20260328-bdc6273`) rather than stable releases. The story spec referenced stable 55.0.6.
**Impact:** Minor — canary versions are functional. React 19.2.0 was installed but some peer deps (`react-native-web`, `@react-aria`) still expect React 18.
**Lesson:** When writing stories, note that pinned version numbers may not match what's available at implementation time. Specify version ranges or "latest SDK 55" instead of exact versions.
**Action:** Monitor for Expo SDK 55 stable release; update when available.

### 2. Gluestack UI v3 + React 19 Compatibility
**Issue:** `@gluestack-ui/core` v3 transitively depends on `@react-aria` which expects React 18. This causes `react-dom@18.3.1` to be pulled in alongside React 19.2.0.
**Impact:** Tests needed mocks for button creator to avoid version mismatch errors. Runtime Expo is unaffected (Metro handles this differently).
**Lesson:** When specifying UI framework dependencies, verify peer dependency chains against the target React version.
**Action:** Add pnpm overrides for react-dom if needed, or wait for @react-aria React 19 support.

### 3. pnpm Not in PATH
**Issue:** pnpm was installed via corepack but not in the default shell PATH — required using full path `/opt/homebrew/bin/pnpm`.
**Lesson:** Document exact pnpm installation method in README for team consistency.

### 4. ESLint v10 Flat Config
**Issue:** ESLint v10 uses flat config (`eslint.config.js`) which drops the `--ext` flag. All workspace `package.json` lint scripts needed updating from `eslint . --ext .ts,.tsx` to `eslint .`.
**Lesson:** When upgrading tools, cascade the changes to all workspace configs in the same story.
